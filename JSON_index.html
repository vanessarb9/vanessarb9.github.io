¿Qué es JSON?

JSON significa Notación de Objeto JavaScript en español o en ingles JavaScript Object Notation JSON es un formato de datos basado en objetos que utiliza una sintaxis similar a la de los objetos en JavaScript. Este formato permite representar estructuras de datos como arreglos y objetos en JavaScript. JSON es un formato de datos ligero y fácil de leer y escribir.

  • Formato: El formato JSON es un texto plano.
  • Sintaxis: Los datos se representan como pares clave-valor separados por comas, y los pares se encierran entre llaves {}.
  • Ligero: JSON es un formato de datos más ligero que XML, lo que puede permitir su uso en aplicaciones más ligeras y en el navegador.
  • Estandar: JSON es un estándar de la W3C.
  • Utilización: JSON puede ser utilizado para representar estructuras de datos en JavaScript, como arreglos y objetos.

Usos del JSON

  • Representar estructuras de datos en JavaScript.
  • Comunicarse con APIs y APIs RESTful.
  • Serializar y deserializar datos.
  • Almacenar y recuperar datos en el navegador.
  • Crear aplicaciones de escritorio y móviles.

    Características de JSON

    • Los valores pueden ser numéricos, cadenas, objetos, arreglos, booleanos, o null.
    • Los valores pueden estar anidados, es decir, un objeto puede contener otro objeto como valor.
    • Los valores pueden estar en una lista, es decir, un arreglo puede contener otros valores como elementos.
    • Los valores no pueden ser objetos literales, es decir, no se puede usar una función como valor.

    Ventajas y desventajas de JSON

    • Ventajas:
      • Ligero y fácil de leer y escribir.
      • Utilizable en aplicaciones más ligeras y en el navegador.
      • Estándar de la W3C.
    • Desventajas:
      • Menos descriptivo: puede que sea facil de leer, pero lo negativo es que no proporciona tanta Información como XML lo hace, lo cúal es limitante.
      • Carece de una forma natica de incluir comentarios en su estructura.
      • cuando se trata de datos no estructurados o semiestructurados, como documentos complejos o contenido multimedia, JSON puede resultar menos eficiente. En tales casos, XML o formatos más flexibles podrían ser opciones más adecuadas.
